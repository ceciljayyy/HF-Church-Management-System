Church Management System
MVP Scope and Build Brief
A focused first version of the full church administration platform
MVP Priority: build a stable, login-ready, database-backed system before adding
advanced modules.

data, and manage the core church operations needed for a first usable release.

Architecture Rules
?
The frontend must not access the database directly.
?
The backend is the only layer that uses Prisma.
?
The frontend must call backend API routes through a typed API client.
?
Use clean feature-based folders; do not put everything in one giant file.
?
Use validation, error handling, loading states, and empty states.
?
Use pagination, search, and filters for large lists.
?
Important create, update, delete, and finance actions must write to AuditLog.
